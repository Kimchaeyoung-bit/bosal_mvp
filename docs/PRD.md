# 강남보살 (Gangnam Bosal) — 서비스 기획서

## 1. 앱 개요

- **앱명(가칭)**: 강남보살
- **서비스 정체성**: 지역 기반으로 실제 활동 중인 보살과 사용자를 연결하고, 챗봇 상담을 통해 나에게 맞는 보살을 추천·예약까지 이어주는 플랫폼
- **벤치마킹**
  - 강남언니: 지역 기반 + 전문가 정보 + 예약 UX
  - 쏘카: 위치 중심 선택 구조 + 간결한 플로우

---

## 2. 기획 배경 & 문제 인식

- 점, 사주, 상담에 대한 수요는 높지만
  - 정보 신뢰도가 낮음
  - 어떤 보살이 나에게 맞는지 알기 어려움
  - 오프라인 접근성이 떨어짐
- 기존 플랫폼은 광고성 정보 위주, 사용자 중심 추천 부족

> **"보살 선택의 기준을 만들어주는 플랫폼"** 필요

---

## 3. 타겟 유저

### 1차 타겟
- 20~40대 여성
- 연애, 진로, 인간관계 고민
- 점·사주 경험은 있으나 만족도가 들쭉날쭉한 사용자

### 2차 타겟
- 지방 거주자
- 검증된 보살을 찾기 어려운 사용자

---

## 4. 핵심 가치 제안

1. **나에게 맞는 보살 추천** — 챗봇 기반 상담 니즈 분석
2. **실제 활동 중인 보살 연결** — 지역·활동 정보 투명화
3. **예약까지 한 번에** — 탐색 → 상담 → 예약 원스톱

---

## 5. 주요 기능 구성

### 5-1. 메인 플로우 A — 지역 선택 기반 ✅ (MVP 우선 구현)

1. **지역 선택** (지도 / 리스트)
2. **근처 보살 리스트 노출**
   - 거리
   - 전문 분야 (연애, 재물, 진로 등)
   - 상담 스타일 (직설 / 공감 / 현실 조언)
3. **보살 상세 페이지**
   - 한 줄 소개
   - 상담 키워드
   - 이용자 후기 요약
4. **챗봇 사전 대화**
   - 사용자의 고민 입력
   - 보살 스타일 적합도 안내
5. **예약 확정**

### 5-2. 메인 플로우 B — 챗봇 추천 기반

1. 챗봇 시작 — "어떤 고민이 있으신가요?"
2. 챗봇 분석 결과 — 추천 보살 2~3명 제안
3. 보살 선택 → 예약

---

## 6. 보살 프로필 구조

| 항목 | 내용 |
|------|------|
| 이름 / 활동명 | 활동명 기준 표시 |
| 활동 지역 | 시/구 단위 |
| 상담 경력 | 연차 |
| 전문 분야 태그 | 연애 / 재물 / 진로 등 |
| 상담 스타일 태그 | 직설 / 공감 / 현실 조언 |
| 실제 공간 사진 | 인증 사진 |
| 평균 만족도 | 별점 X → 키워드 기반 |

---

## 7. 수익 모델

- **사용자**: 앱 이용 및 예약 무료
- **수익원**
  1. 보살 유료 입점 / 노출 광고
  2. 브랜드 협업 콘텐츠 (힐링, 명상, 운세)
  3. 추천 보살 상단 노출 광고
  4. (확장) 온라인 상담 중개 수수료

---

## 8. 신뢰 & 안전 설계

- 보살 인증 프로세스 (신분 확인, 활동 사진 제출)
- 신고 / 후기 시스템
- 챗봇 상담 기록 비공개

---

## 9. MVP 구현 범위

### 1차 MVP
- 지역 선택
- 보살 리스트 / 상세
- 챗봇 추천 (룰 기반)
- 예약 요청

### 2차 확장
- AI 추천 고도화
- 후기 분석 자동 요약
- 비대면 상담 연결

---

## 10. 지역 선택 기반 플로우 (확정 디자인 아님)

```
[메인] → [지역 선택] → [보살 리스트] → [보살 상세] → [챗봇 사전 대화] → [예약 확정]
```

본 MVP에서는 위 플로우의 진입점인 **메인 페이지**를 우선 제작한다.
